■ ERD (ENTITY  RELATIONSHIP DIAGRAM)    
-> 데이터 관계간에 초점을 둔 모델
1. 개체 (Entity) : 테이블
2. 속성 (Attribute) : 열, 컬럼
3. 관계 (Relationship) : 외래키

■ 정규화 (Normal Format)
-> 관계형 데이터 모델의 이상(삽입, 삭제, 갱신)을 제거하는 것
    1NF - 중복항목 없애기
    2NF - 종속 관계 X (결정자:PK) -> Y (종속자) 기본키에 완전 함수 종속
    3NF - 테이블내의 모든 속성이 기본키에만 의존해야 하며 다른 후보키에 의존하지 않음
	before) X -> Y -> Z
	after) X -> Y  /  Y -> Z
    BCNF - 모든 결정자 후보키
    4NF - 다치 종속 / 교수2 : 교재2 
    5NF - 조인 종속 / 다 : 다 관계 -> 중간 테이블 필요
